## Список реализованных проверок

### 1. Успешная авторизация
- **Метод:** `POST /auth/login`
- **Что проверяет:** Авторизация с валидными данными (`username: emilys`, `password: emilyspass`) возвращает `accessToken`
- **Ожидаемый результат:** Статус-код 200, в ответе присутствует поле `accessToken`

### 2. Неуспешная авторизация
- **Метод:** `POST /auth/login`
- **Что проверяет:** Авторизация с неверным паролем возвращает ошибку
- **Ожидаемый результат:** Статус-код 400, в ответе присутствует сообщение об ошибке

### 3. Получение пользователя с токеном
- **Метод:** `GET /auth/me`
- **Что проверяет:** Получение данных пользователя с валидным токеном авторизации
- **Ожидаемый результат:** Статус-код 200, в ответе присутствуют данные пользователя (`id`, `username`, `email`)

### 4. Получение пользователя без токена
- **Метод:** `GET /auth/me`
- **Что проверяет:** Запрос данных пользователя без передачи токена
- **Ожидаемый результат:** Статус-код 401 (Unauthorized), доступ запрещён

### 5. Получение корзин пользователя
- **Метод:** `GET /carts/user/{userId}`
- **Что проверяет:** Получение списка всех корзин для конкретного пользователя
- **Ожидаемый результат:** Статус-код 200, в ответе присутствует массив `carts`

### 6. Получение корзины по ID
- **Метод:** `GET /carts/{cartId}`
- **Что проверяет:** Получение конкретной корзины по её идентификатору
- **Ожидаемый результат:** Статус-код 200, ID корзины в ответе совпадает с запрошенным

### 7. Создание корзины
- **Метод:** `POST /carts/add`
- **Что проверяет:** Создание новой корзины с указанными товарами
- **Ожидаемый результат:** Статус-код 200 или 201, в ответе присутствует новый `id`

### 8. Обновление корзины
- **Метод:** `PUT /carts/{cartId}`
- **Что проверяет:** Добавление новых товаров в существующую корзину (с параметром `merge: true`)
- **Ожидаемый результат:** Статус-код 200, общее количество товаров увеличивается

### 9. Удаление корзины
- **Метод:** `DELETE /carts/{cartId}`
- **Что проверяет:** Удаление корзины (симуляция)
- **Ожидаемый результат:** Статус-код 200, в ответе присутствуют `isDeleted: true` и `deletedOn`

### 10. Негативный тест (несуществующая корзина)
- **Метод:** `GET /carts/999999`
- **Что проверяет:** Запрос корзины с несуществующим ID
- **Ожидаемый результат:** Статус-код 404, в ответе присутствует сообщение об ошибке

## Известные проблемы

### Проблема 1: Статус-код при создании корзины
Описание: В документации DummyJSON указано, что POST /carts/add возвращает статус 200, но в реальности API возвращает статус 201 (Created). Это более корректное поведение, соответствующее HTTP-стандартам.
Решение: Тест адаптирован для обоих статусов (200 и 201).
Статус: Исправлено.

### Проблема 2: Симуляция операций с корзиной
Описание: API DummyJSON не сохраняет изменения на сервере при создании, обновлении и удалении корзины. Все операции являются симуляцией и возвращают моковые данные.
Ожидаемое поведение: Для тестовых целей это приемлемо, так как API предназначен для prototyping.
Статус: Принято как особенность API.
